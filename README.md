# 🎮 Авто-Telegram канал игровых новостей

Полностью бесплатный бот на **GitHub Actions + Python**: собирает игровые новости из десятков RSS-источников, бесплатный ИИ (Gemini / OpenRouter) выбирает топ-5 и пишет красивые посты на русском с картинкой, и публикует их в канал. Сервисный бот шлёт тебе отчёты в ЛС и принимает команды «по запросу».

## 📁 Структура проекта

```
gaming-news-bot/
├── .github/workflows/
│   ├── curate.yml      # 1x в день: сбор новостей + выбор топ-5 ИИ (1 вызов ИИ)
│   ├── publish.yml     # 5 раз в день: публикация по одному посту
│   └── bot.yml         # каждые 30 мин: обработка команд сервисного бота
├── src/
│   ├── __init__.py
│   ├── config.py       # все настройки из переменных окружения (Secrets)
│   ├── sources.py      # список RSS-источников
│   ├── fetcher.py      # парсинг RSS за 24ч + поиск картинки (og:image)
│   ├── ai.py           # выбор топ-5 и генерация постов (Gemini -> OpenRouter)
│   ├── telegram.py     # публикация в канал + отчёты (HTML parse_mode)
│   ├── state.py        # файловое состояние (очередь / опубликованное / offset)
│   ├── bot.py          # команды «по запросу» (/post, /batch, /status)
│   └── main.py         # точка входа: curate | publish | publish_all | bot
├── data/               # состояние (коммитится ботом обратно в репо)
│   ├── posted.json     # []
│   ├── queue.json      # []
│   └── offset.json     # {"offset": 0}
├── requirements.txt
├── .env.example
└── README.md
```

## ⚙️ Как это работает

1. **curate** (1 раз в день) — собирает новости за 24ч, ИИ выбирает топ-5 и пишет посты, очередь падает в `data/queue.json`. **Всего 1 вызов ИИ в сутки** — легко в бесплатный лимит.
2. **publish** (5 слотов в день) — берёт по одному посту из очереди и публикует. После каждого поста — отчёт в ЛС.
3. **bot** (каждые 30 мин) — слушает твои команды сервисному боту и запускает публикацию «по запросу».

Состояние хранится в `data/*.json` и коммитится обратно в репозиторий — так оно переживает запуски (GitHub Actions без внешней БД).

## 🚀 Быстрый старт

### 1. Создай канал и двух ботов (через @BotFather)
- **Канал** — создай Telegram-канал (публичный с @username или приватный).
- **Канальный бот** — создай бота, добавь его **админом канала** (с правом публикации). Его токен → `CHANNEL_BOT_TOKEN`.
- **Сервисный бот** — отдельный бот для отчётов в ЛС и команд. Его токен → `SERVICE_BOT_TOKEN`. Нажми у него **Start**, чтобы он мог писать тебе.

### 2. Узнай ID
- `CHANNEL_ID` — для публичного канала это `@username`; для приватного — числовой ID вида `-100xxxxxxxxxx`.
- `ADMIN_CHAT_ID` — твой личный chat_id (например, через @userinfobot).

### 3. Получи бесплатные ИИ-ключи
- **Gemini** (основной): https://aistudio.google.com/app/apikey → `GEMINI_API_KEY`.
- **OpenRouter** (резерв, опционально): https://openrouter.ai/keys → `OPENROUTER_API_KEY`.

### 4. Залей код в GitHub и пропиши Secrets
Репозиторий → **Settings → Secrets and variables → Actions → New repository secret**:

| Secret | Обяз. | Описание |
|---|---|---|
| `CHANNEL_BOT_TOKEN` | ✅ | Токен бота-админа канала |
| `CHANNEL_ID` | ✅ | @username или -100... ID канала |
| `SERVICE_BOT_TOKEN` | ✅ | Токен сервисного бота (отчёты/команды) |
| `ADMIN_CHAT_ID` | ✅ | Твой chat_id для отчётов |
| `GEMINI_API_KEY` | ✅ | Ключ Gemini API |
| `OPENROUTER_API_KEY` | ➖ | Ключ OpenRouter (резерв) |

> Необязательные переменные (есть значения по умолчанию): `GEMINI_MODEL`, `OPENROUTER_MODEL`, `POSTS_PER_DAY`, `LOOKBACK_HOURS`, `MAX_CANDIDATES`.

### 5. Включи Actions
Вкладка **Actions** → разреши workflow. Дальше всё работает по cron. Кнопка **Run workflow** (workflow_dispatch) — ручной запуск любого из трёх workflow.

## 🕐 Расписание (cron в UTC!)
Обрати внимание: GitHub Actions использует **UTC**. Твой пояс — UTC+5, поэтому:
- `curate`: `0 5 * * *` → ~10:00 утра по твоему времени.
- `publish`: `0 6,9,12,15,18 * * *` → ~11:00, 14:00, 17:00, 20:00, 23:00.
Меняй цифры в cron под свой часовой пояс.

## 💬 Команды «по запросу» (в сервисном боте)
- `/post` — опубликовать одну новость прямо сейчас (если очередь пуста — сначала соберёт).
- `/batch` — собрать и опубликовать сразу всю пачку.
- `/status` — сколько постов в очереди.

> Команды обрабатываются раз в ~30 минут (cron `bot.yml`). Хочешь мгновенно — нажми **Run workflow** у `bot.yml` или `publish.yml` во вкладке Actions.

## 💰 Бесплатность
- **Публичный репозиторий** → GitHub Actions безлимитны (рекомендуется).
- **Приватный** → 2000 минут/мес бесплатно. Основной расход — опрос бота каждые 30 мин. Можно увеличить интервал в `bot.yml` или отключить его cron и жать Run workflow вручную.
- **ИИ**: всего 1 вызов в сутки (curate) — комфортно в бесплатном лимите Gemini.

## 🧪 Локальный запуск
```bash
pip install -r requirements.txt
cp .env.example .env   # заполни значения
set -a && source .env && set +a
python -m src.main curate        # собрать и сформировать очередь
python -m src.main publish       # опубликовать 1 пост
python -m src.main publish_all   # опубликовать всю пачку
python -m src.main bot           # обработать команды
```

## 🔧 Кастомизация
- **Источники**: редактируй `src/sources.py` — добавляй/убирай RSS-ленты.
- **Стиль постов**: правь промпт `PROMPT` в `src/ai.py`.
- **Частота/количество**: `POSTS_PER_DAY` + cron в `publish.yml`.
